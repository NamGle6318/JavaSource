Thread : 
    - 프로세스 : 현재 실행중인 프로그램
    - 쓰레드 : 하나의 프로세스 안에 하나 이상의 Thread 사용 
               실제 프로그램이 수행되는 최소 작업 단위
멀티 쓰레드 작성 
 - Thread 클래스 상속 - 다른 클래스를 상속받지 못함
 - Runnable interface 사용 

 - 쓰레드를 실행하면 스케쥴러에 따라 번갈아가면서 실행

cpu 싱글 코어 : 스레드 우선순위에 따라 우선 순위 높은애가 돌릴 기회가 더 많음
cpu 쿼드 코어 : 4가지일을 동시에 할 수 있음.

동기화 메소드와 동기화 블록
- 멀티스레드 프로그램에서는 스레드들이 객체를 공유해서 작업하는 경우가 있음
- 공유해서 사용하는 경우 한 스레드가 먼저 사용중이라면 다른 스레드의 접근을 막아야함
- = 임계영역 지정 => synchronized  : 한번에 하나의 스레드만 접근 가능

 start()
 run() - 실행대기
 join()
 sleep(ms)
 intarrupt()
 synchronized : 경쟁조건 방지(중복 실행)
 ExecutorService : 쓰레드 풀(쓰레드 재사용하여 자원 절약)
