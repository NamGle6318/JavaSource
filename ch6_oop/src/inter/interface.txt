인터페이스
- 추상클래스(단 추상화 정도가 높다)
- 일반메서드, 멤버 변수를 가질 수 없음 ( 상수는 선언 가능)
- 인터페이스 안에 들어올 수 있는 것 (상수, 추상 메서드, (static, default) 메서드)
- 상수밖에 선언할 수 없기 때문에 선언시 public static final 생략 가능
- 추상메서드만 선언 가능하기 때문에 선언시 public abstract 을 생략할 수 있다.
- 인스턴스 직접 생성 불가능 (new) 참조하고 있는 인스턴스를 생성해야한다. (다형성)
- 인터페이스 상속 : - 인터페이스로만 상속을 받는다. / 다중상속
- 장점 : 표준화가 가능하다 / 서로 관계없는 클래스들에게 관계를 맺어 줄 수 있다.
        /의존관계를 약화시킬 수 있다.

 : 클래스는 단일상속만 가능 
 class A extends B {}
 다중 상속
 class A implement B, C, D, E, F, G, H, I, J, K, L {}
